## Observations worth flagging (cosmetic / not blocking)

1. **Blockquote `>` is escaped to `&gt;` by Plane on write** — visible in `description_html`. Visual rendering is unaffected (Plane shows `>` as text either way), but the **`/migrate-reassign` parser must anchor on both `> **Migrated from Jira` and `&gt; **Migrated from Jira`** when reading existing descriptions. Flag for `/migrate-implement reassign`.
2. **URLs in the custom-fields footer are wrapped in `<…>` (markdown autolink syntax)** which Plane's HTML normaliser escapes to `&lt;…&gt;`. URL text is still visible/copyable; clickable hyperlinking is lost. Same trade-off as LDRH — acceptable for now.
3. **6 issues were created on the Plane project before the migration** (existing Plane test data wasn't on this project, but the LRP Jira project `LRP-1` and 5 others were already accounted as missing — Plane sequence_id starts at 1, matching). For LRP, the Plane sequence drift is `Jira LRP-2 → Plane LRP-1` (sequence_id offset = 1) because Plane assigns sequential IDs on creation rather than honouring Jira's original numbering. Not a failure; just note that the **Plane identifier ≠ Jira identifier**.
